Uso

A diferencia de las odas de primaria, donde la url de carga era apuntada hacia la carpeta de la oda, en las odas de bachillerato se apuntará hacia el root y se pasará el parámetro "oda" con el id a cargar.

El dominio de desarrollo es https://bach.ml/, por lo que todos los ejemplos incuirán esa url.

ODA ALG1P1B

Se pasa como parámetro get oda y como valor el ID de la ODA.

<iframe src="https://bach.ml/?oda=ALG1P1B">

Cada interacción del usuario genera un objeto que se almacena en "localStorage" y se envía por postMessage (interactuar con oda para ver valores en tiempo real) 👇🏽:

{{response}}

▶️ Status: Almacena todas las respuestas del usuario y la ubicación en la navegación.

▶️ Results: Regresa null si no se ha finalizado la actividad.

Cuando se da click en "finalizar" el valor Results devuelve los siguientes campos:

👉 total: Total de preguntas en la oda.

👉 oks: Respuestas correctas.

👉 errors: Respuestas incorrectas.

👉 oksPercentage: Porcentaje de respuestas correctas (0 - 100).

👉 errorsPercentage: Porcentaje de respuestas incorrectas (0 - 100).

Para regresar el valor de Status y cargar las respuestas que lleva el usuario se coloca en el parámetro status: 

<iframe src="https://bach.ml/?oda=ALG1P1B&status={{statusExample}}">

El código javascript para obtener el postMessage es el siguiente:

window.addEventListener('message', function (event) {
 if(typeof event.data == 'string'){
   var mensaje = JSON.parse(event.data)
 }
}, true)

Variantes

Sin ODA ❌

<iframe src="https://bach.ml/">

ODA ALG1P1B Modo bloqueado 👨‍🏫 👩‍🏫 👨 🧑

Si se añade el parámetro blocked se deshabilitan los módulos y botones para reiniciar y finalizar.

Se convierte en un documento sólo de consulta.

Sería el modo para mostrarse a docentes y padres.

<iframe src="https://bach.ml/?oda=ALG1P1B&blocked">

ODA ALG1P1B Modo de pruebas 🕹️

Si se añade el parámetro test se habilita el entorno de pruebas, que muestra botones (amarillos) para resolver las actividades automáticamente.

<iframe src="https://bach.ml/?oda=ALG1P1B&test">
